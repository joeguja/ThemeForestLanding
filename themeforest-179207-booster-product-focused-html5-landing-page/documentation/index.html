HTML structure

Booster's HTML code is properly commented with HTML comments. We believe this will significantly increase your productivity while you make changes to the template. We've marked the beginning and the end of all important elements so that you can easily modify or remove them without compromising the HTML's integrity and validity.
For production environments we recommend removing the HTML comments. If your HTML editor of choice can't remove them automatically you can easily clean your code using one of many free online tools such as HTML Tidy Online.

C) JavaScript - top

All Booster's JS code is jQuery based. Here's the list of all JS files. They're located in the JS folder.

  • jquery-1.5.min.js - main jQuery library, has to be included on all pages
  • modernizr-1.6.min.js - HTML5 compatibility fix for older browsers
  • jquery.form.2.64.js - form and AJAX handling for newsletter subscription
  • jquery.validate.js - form validation
  • jquery.prettyPhoto.js - prettyPhoto lightbox script
  • jquery.tipsy.js - tipsy tooltips script
  • twitter.min.js - twitter script
  • common.js - misc code, init and config for all scripts


Configuring the newsletter subscription script
If you're having problems with the newsletter subscription script please read the 5 seconds PHP setup first. There's no need to change anything in JS unless you want to change the response script gives to the user.
Open "common.js" file which you'll find in the JS directory. The code you need starts on line 80. You can change the single-quoted text in each line that starts with "alert". For instance if you change the text on line 96 ("Thank you for subscribing to our newsletter! We will not abuse your address.") you'll change the "thank you" text users see after a successful subscription.


Configuring Tipsy tooltips
You'll find the tipsy init scrips online #45 in common.js. There are only 3 variables to configure:

  • delayIn - time it takes the tooltip to appear after you mouse-in
  • delaOut - time it takes the tooltip to disappear after you mouse-out
  • gravity - tooltips position, relative to parent element; options: n, s, e, w


Configuring prettyPhoto lightbox
You'll find the prettyPhoto init scrips online #41 in common.js. All variables are self explanatory. The script targets all images with rel="prettyPhoto" that are inside ".gallery" div.
Configuration variables that you'll want to change are:

  • animation_speed - fast/slow/normal
  • slideshow - false OR interval time in ms
  • autoplay_slideshow - true/false
  • opacity - Value between 0 and 1
  • show_title - true/false
  • allow_resize: - Resize the photos bigger than viewport. true/false
  • default_width - 500; change if necessary
  • default_height - 344; change if necessary
  • theme - light_rounded, dark_rounded, light_square, dark_square or facebook
  • autoplay - Automatically start videos: True/False
  • modal - If set to true, only the close button will close the window
  • overlay_gallery - If set to true, a gallery will overlay the fullscreen image on mouse over
  • keyboard_shortcuts - Set to false if you open forms inside prettyPhoto

Read PrettyPhoto documentation for more detailed options info.


Configuring the "drop us a line" form contact script
If you're having problems with the contact form please read the 5 seconds PHP setup first. There's no need to change anything in the JS/HTML unless you want to add/remove form elements.
Validation and AJAX form handling is defined and configured in "common.js". We've used one of the best validation scripts available - jQuery validation plugin which can be easily configured to validate any form. Lines 66-73 in "common.js" (rules) define the rules for input fields. Most of them are only "required" while the email field also needs to be a valid email address. Lines 102-109 (messages) define the error messages user gets when the form is not properly filled.
Once the form is submitted via AJAX response alerts are configured in the "contactFormResponse()" function. You'll probably want to change the message for successful sending which is defined at line 112.


Configuring the Twitter script
Open "common.js" file which you'll find in the JS directory. On line 53 there's a piece of code that contains "id: 'envatowebdesign',". Simply replace "envatowebdesing" with the Twitter username who's tweets you want to show and you're done! Don't forget the single quotes around the username.
If you have some time you can further customize the script. Variable "count" defines how many tweets you want to show in your template. "enableLinks" enables/disables links that are a part of the tweet itself. "ignoreReplies" defines if you (don't) want to skip tweets starting with @. And finally "template" defines how each tweet is outputted to your template.
For further details please see code.google.com/p/twitterjs/ and remysharp.com/2007/05/18/add-twitter-to-your-blog-step-by-step/.

D) PHP Files - top

"Drop us a line" contact form

Booster includes a versatile AJAX driven PHP script that powers the "drop us a line" form.

5 seconds setup
Locate the file "contact-us.php" which you'll find in the root directory of your chosen color theme. On line 9 there's a variable called $myEmail fill it with your email address, ie "$myEmail = 'john@gmail.com';". You're done!
Test the script by entering some data into the contact form and you should receve an email momentarily.

5 minutes setup
Besides the email address you can also configure the email's subject. You'll find the variable "$emailSubject" online 13 of "contact-us.php" script.
If you add or remove some form fields there is no need to change anything in the script. It will email you all the fields regardless of their number.
All other parameters of this script are configured in JavaScript. You can read more about it in the JavaScript section of this manual.

Mailchimp newsletter signup script

In order to activate the MailChimp newsletter script you have to change the "action" property of the newsletter form from "newsletter-subscribe.php" to "newsletter-subscribe-mailchimp.php".

Next step is to configure the script itself. Open "newsletter-subscribe-mailchimp.php". Only change lines #9 and #11. On line #9 enter your MailChimp API key (don't forget the quotes). On line #11 enter the list name you want people to subscribe to, again - don't forget the quotes.

Newsletter signup script

Booster includes a simple AJAX driven PHP script that enables you to collect email addresses from your visitors who whish to subscribe to your newsletter.

Script requirements
PHP 5.2 or an older version of PHP with JSON extension. Privileges to modify write-access (chmod) on one txt file. Ability to send emails via PHP's "mail()" function.
These requirements are met by default in almost all hosting environments that support PHP.

5 seconds setup
Locate the file "subscribed-emails.txt" which you'll find in the root www directory of template. Chmod it to 777 so that PHP can write the emails in it. You're done!
Test the script by entering an email in "subscribe" box on your site, hit enter and it should say "Thank you for subscribing".

5 minutes setup
All PHP code is located in a single file "newsletter-subscribe.php". There are 2 variables that you can configure. The first one, "$emailsFile" (line 10) defines the path to TXT file in which PHP stores the emails and checks for duplicates. PHP has to have read and write access to this file so be sure to chmod it to 777 or a similar value that will give PHP the noted access rights. Second variable "$myEmail" (line 15) is an optional parameter which enables you to receive an email each time someone subscribes toyour newsletter. Set this variable to your email address, ie: "myemail@gmail.com". If you don't want to use this feature just leave it blank.
All other parameters of this script are configured in JavaScript. You can read morea bout it in the JavaScript section of this manual.
